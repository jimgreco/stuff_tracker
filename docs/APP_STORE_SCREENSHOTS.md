# App Store Screenshots

Screenshots are generated by a dedicated UI-test target and a debug-only launch mode. The app runs offline, clears auth state, seeds deterministic local data, and captures the real SwiftUI screens through Fastlane.

## Generate Locally

```sh
bundle install
bundle exec fastlane ios screenshots
```

Screenshots are written to `fastlane/screenshots/`. To override the default devices:

```sh
SCREENSHOT_DEVICES="iPhone 17 Pro Max,iPad Pro 13-inch (M5)" bundle exec fastlane ios screenshots
```

## Upload To App Store Connect

```sh
APP_STORE_CONNECT_KEY_ID=... \
APP_STORE_CONNECT_ISSUER_ID=... \
APP_STORE_CONNECT_API_KEY="$(cat AuthKey_XXXXXX.p8)" \
bundle exec fastlane ios upload_screenshots
```

The upload lane targets the checked-in Xcode `MARKETING_VERSION` for `StuffTracker` unless `APP_STORE_VERSION` is set. It uploads screenshots only: metadata and binary upload are skipped, precheck is skipped, and existing screenshots for that editable version are replaced.

To capture and upload in one command:

```sh
APP_STORE_CONNECT_KEY_ID=... \
APP_STORE_CONNECT_ISSUER_ID=... \
APP_STORE_CONNECT_API_KEY="$(cat AuthKey_XXXXXX.p8)" \
bundle exec fastlane ios screenshots_and_upload
```

## GitHub Actions

Run **App Store Screenshots** from the Actions tab.

- Leave `upload_to_app_store` as `false` to generate a review artifact only.
- Set `upload_to_app_store` to `true` after reviewing the artifact to replace screenshots in App Store Connect.
- Use `app_store_version` only when App Store Connect needs a version different from the checked-in Xcode `MARKETING_VERSION`.
- The workflow uses the same `APP_STORE_CONNECT_KEY_ID`, `APP_STORE_CONNECT_ISSUER_ID`, and `APP_STORE_CONNECT_API_KEY` secrets as the TestFlight workflow.
